# Notification Outbox

Gatherly использует outbox-like подход для in-app уведомлений.

Когда происходит важное доменное действие, сервис сохраняет намерение отправить уведомление в `notification_outbox`. В production runtime это сохраняется в PostgreSQL через `notification_outbox`. Для MVP операции выполняются последовательно в сервисном слое; следующим шагом их можно объединить в явные SQL-транзакции.

## Зачем это нужно

- Публикация события не должна откатываться, если канал уведомлений временно сломан.
- Failed records можно повторить позже.
- Дизайн демонстрирует reliable event processing и graceful degradation.

## Режимы

- `NORMAL`: outbox records сразу превращаются в in-app уведомления.
- `FAILING`: обработка падает, records получают статус `FAILED`.
- `RECOVERY`: pending и failed records повторно обрабатываются и доставляются.

## Поведение в MVP

Worker запускается вместе с сервисом и периодически повторяет pending/failed records. Наружу отдаются только пользовательские endpoints уведомлений: список моих уведомлений и отметка уведомления прочитанным.
