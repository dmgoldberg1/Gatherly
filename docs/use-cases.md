# Use Cases

## Регистрация пользователя

1. Пользователь отправляет username, email и password.
2. Система проверяет обязательные поля и уникальность.
3. Система создаёт пользователя и возвращает профиль.

## Подписка на другого пользователя

1. Bob авторизован.
2. Bob подписывается на Alice.
3. Система запрещает self-follow и повторную подписку.
4. Будущие опубликованные события Alice могут попасть в feed Bob.

## Создание и публикация события

1. Alice создаёт draft-событие.
2. Alice при необходимости обновляет детали draft.
3. Alice публикует событие.
4. Система создаёт notification outbox records для подписчиков Alice.
5. Notification processor доставляет in-app уведомления, если канал работает.

## Присоединение к событию

1. Bob открывает видимое опубликованное событие.
2. Bob присоединяется к событию.
3. Система создаёт участие со статусом GOING, если лимит не исчерпан.
4. Если событие заполнено, Bob получает статус WAITLISTED.

## Уведомление об обновлении события

1. Alice меняет время или место события.
2. Система сохраняет изменения.
3. Система создаёт outbox records для участников.
4. Участники получают in-app уведомления после обработки outbox.

## Отказ доставки уведомлений и восстановление

1. Admin переключает notification mode в FAILING.
2. Alice публикует событие.
3. Публикация события проходит успешно, но outbox records получают статус FAILED.
4. Admin переключает режим в RECOVERY.
5. Admin запускает обработку outbox.
6. Failed records повторно обрабатываются, уведомления доставляются.
